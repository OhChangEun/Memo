OSI 7계층(Open Systems Interconnection Model)은 네트워크 통신을 체계적으로 이해하기 위해 정의된 모델로, 
데이터가 한 장치에서 다른 장치로 전달될 때 각 계층이 어떻게 역할을 수행하는지 설명합니다. 
OSI 모델은 7개의 계층으로 나뉘며, 각 계층은 특정한 역할을 담당하고 있습니다. 
계층은 위에서 아래로 내려갈수록 실제 데이터 전송과 관련된 기능을 담당합니다.

1. 물리 계층 (Physical Layer)
역할: 물리 계층은 실제 데이터 전송을 위한 하드웨어적 장치와 신호를 처리합니다. 이 계층은 전기적, 기계적, 절차적 측면에서 데이터 전송 방식을 정의합니다.

주요 기능: 데이터 전송 매체(전선, 광섬유, 무선 주파수 등), 전기적 신호, 비트 전송
예시 장비: 허브, 리피터, 케이블, NIC(Network Interface Card)

2. 데이터 링크 계층 (Data Link Layer)
역할: 데이터 링크 계층은 물리 계층을 통해 전송되는 데이터를 프레임이라는 단위로 나누어 안정적이고 오류 없는 전송을 보장합니다. 또한, 네트워크 장치 간의 주소 지정 및 흐름 제어를 담당합니다.

주요 기능: 오류 감지 및 수정, 프레임 형성, MAC 주소 사용
예시 장비: 스위치, 브리지

3. 네트워크 계층 (Network Layer)
역할: 네트워크 계층은 패킷을 목적지까지 전달하는 경로를 선택하고, 여러 네트워크 간의 데이터 전송을 처리합니다. 이 계층에서는 IP 주소를 사용하여 데이터의 출발지와 목적지를 식별합니다.

주요 기능: 라우팅, IP 주소 지정, 패킷 전송
예시 장비: 라우터

4. 전송 계층 (Transport Layer)
역할: 전송 계층은 데이터가 목적지 시스템에서 정확하게 수신될 수 있도록 보장하며, 에러 복구와 흐름 제어를 담당합니다. 또한, 데이터의 구분 및 순서 지정을 제공합니다.

주요 기능: 세그먼트화, 오류 복구, 흐름 제어, 포트 번호를 이용한 통신
예시 프로토콜: TCP, UDP

5. 세션 계층 (Session Layer)
역할: 세션 계층은 네트워크에서의 세션 관리를 담당하며, 세션 시작, 유지, 종료를 제어합니다. 즉, 통신하는 두 시스템 간에 대화식 연결을 설정하고, 중단된 연결을 다시 시작할 수 있도록 합니다.

주요 기능: 세션 설정 및 관리, 동기화
예시 프로토콜: NetBIOS, RPC (Remote Procedure Call)

6. 표현 계층 (Presentation Layer)
역할: 표현 계층은 데이터의 형식 변환을 담당합니다. 데이터가 송신자와 수신자 간에 호환되지 않는 형식으로 전달될 경우 이를 변환, 암호화, 압축합니다.

주요 기능: 데이터 인코딩, 데이터 압축, 데이터 암호화
예시: JPEG, GIF, SSL/TLS

7. 응용 계층 (Application Layer)
역할: 응용 계층은 사용자와 네트워크 간의 인터페이스를 제공합니다. 이 계층에서는 실제 애플리케이션과 관련된 프로토콜들이 동작하며, 사용자가 네트워크 서비스를 이용할 수 있도록 합니다.

주요 기능: 데이터 전송 및 요청 처리, 애플리케이션 서비스 제공
예시 프로토콜: HTTP, FTP, SMTP, DNS

요약

1계층 (물리 계층): 실제 신호 전송 (비트 전송)
2계층 (데이터 링크 계층): 프레임 전송 및 오류 수정 - 1대1 통신: 내가 보낸 데이터를 상대가 받을 수 있으면
3계층 (네트워크 계층): 패킷 전송 및 라우팅
4계층 (전송 계층): 세그먼트화 및 데이터 오류 처리
5계층 (세션 계층): 통신 세션 관리
6계층 (표현 계층): 데이터 형식 변환, 암호화
7계층 (응용 계층): 사용자와 네트워크 간의 인터페이스
