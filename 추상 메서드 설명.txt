**추상 메서드(abstract method)**는 
메서드의 이름, 매개변수, 반환형만 있고 본체(=중괄호 {} 내부의 코드)는 없는 메서드입니다.
즉, **"무엇을 해야 하는지는 정의되어 있지만, 어떻게 할지는 정해져 있지 않은 메서드"**입니다.

✅ 추상 메서드란?
abstract 키워드를 사용함
**중괄호 {} 없이 세미콜론 ;으로 끝남
반드시 추상 클래스나 인터페이스 안에서만 선언 가능
추상 메서드를 가진 클래스는 반드시 추상 클래스여야 함

✅ 예시 (Java)
abstract class Animal {
    // 추상 메서드
    public abstract void makeSound();

    // 일반 메서드
    public void breathe() {
        System.out.println("숨을 쉽니다.");
    }
}
위 코드에서 makeSound()는 추상 메서드입니다.
이 메서드는 이름만 있고 실제 동작 내용은 없습니다.
그래서 이 메서드를 사용하는 자식 클래스가 반드시 구현해야 합니다.

✅ 자식 클래스에서 구현
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("멍멍!");
    }
}
Dog는 Animal을 상속받았기 때문에 makeSound()를 반드시 구현해야 합니다.
그렇지 않으면 컴파일 에러 발생

✅ 왜 추상 메서드를 쓸까?
자식 클래스에게 특정 기능 구현을 강제하기 위해
다양한 클래스가 공통된 인터페이스(형식)를 따르도록 유도
프로그램의 일관성과 유지보수성을 높임

✅ 요약
항목	설명
정의	몸체 없이 선언만 된 메서드 (abstract void method();)
위치	추상 클래스나 인터페이스 안
목적	자식 클래스에게 반드시 구현하도록 강제
특징	본문 없음, 반드시 오버라이딩 필요
