
**어노테이션은 함수가 아니고, ‘정보(메타데이터)를 알려주는 특별한 표식’**이에요.

✅ 한 줄 정의
어노테이션은 자바 컴파일러나 프레임워크(Spring 등)에게
“이 코드가 어떤 역할을 하는지 알려주는 표시(메타데이터)”예요.

✅ 비유로 이해해 보기

📌 어노테이션은 명찰 같아요!
예시					의미
@RestController			"이 클래스는 웹에서 데이터를 주고받는 API야!"
@GetMapping("/hello")		"이 함수는 GET 방식으로 /hello 요청 받을 때 실행돼!"
@Id					"이 필드는 데이터베이스에서 기본키야!"
@Autowired			"이 객체는 스프링이 알아서 넣어줄게!"

👉 즉, 어노테이션은 실제로 실행되는 함수는 아니지만,
그 코드에 붙여서 특별한 의미를 부여하는 거예요.

✅ 어노테이션 vs 함수 차이
구분	어노테이션	함수
정의	메타데이터 (설명, 표시)	실제 실행되는 코드
역할	코드의 의미나 동작을 정의	기능을 수행함
예시	@GetMapping, @Entity, @Autowired	public String hello() { ... }

✅ 어노테이션은 언제 실행될까?
어노테이션은 컴파일 시나 실행 시(Spring 같은 프레임워크가)
해당 어노테이션을 보고 "아, 이걸 어떻게 처리해야겠다" 하고 자동으로 동작을 연결해줘요.

예:
@GetMapping("/hello")
public String hello() { return "Hi"; }
→ 여기서 @GetMapping은 메서드를 실행하지 않지만
→ 스프링이 보고 "아, 이 메서드는 /hello 요청이 오면 실행해야겠네!" 하고 연결해주는 거예요.

✅ 정리해서 설명하면
어노테이션은 실행되는 함수가 아니라,
**코드에 의미를 부여하고, 프레임워크가 자동으로 처리하게 도와주는 '표시'**입니다.
쉽게 말해, "이 코드는 이런 역할을 해!"라고 명찰을 붙이는 것이에요.

원하시면 @Autowired, @RequestParam 같은 어노테이션들이
정확히 뭘 의미하는지 예시 중심으로 알려드릴게요! 😊




“명칭만 붙였는데 어떻게 기능이 자동으로 수행될까?” —
이건 자바와 스프링이 제공하는

 "리플렉션(reflection)"과 프레임워크의 자동 처리 덕분이에요.

✅ 핵심 개념 요약
어노테이션은 **단순한 표시(메타데이터)**이고,
스프링이 실행 중에 그 표시를 읽고, 해당 기능을 자동으로 연결해주는 거예요.
이걸 가능하게 하는 기술이 바로 **리플렉션(Reflection)**이에요.

✅ 쉽게 설명하자면...
@RestController
public class HelloController {

    @GetMapping("/hello")
    public String hello() {
        return "Hi";
    }
}

그럼 무슨 일이 일어날까?
Spring이 실행될 때, 프로젝트 전체를 뒤지면서
클래스 위에 @RestController 붙은 걸 찾고
메서드 위에 @GetMapping("/hello") 붙은 것도 찾아요
그리고 이렇게 기록해둡니다:

"아하! /hello 라는 경로가 오면 이 hello() 메서드를 실행해야겠군!"

사용자가 웹브라우저에서 /hello에 접속하면?

Spring이 이 기록을 바탕으로 hello() 메서드를 자동 실행해서 결과를 반환해요.

✅ 기술적으로는 어떻게 가능할까?
자바는 리플렉션 API라는 기능으로, 실행 중인 클래스에서
어떤 어노테이션이 붙었는지
어떤 메서드가 선언됐는지
등의 정보를 읽어낼 수 있어요.

스프링은 이걸 활용해서...
@Controller가 붙은 클래스만 모아서 컨트롤러로 등록하고
@GetMapping("/hello")가 붙은 메서드는 /hello 요청과 연결하고
@Autowired가 붙은 필드는 객체를 자동으로 넣어줘요 (DI, 의존성 주입)

✅ 비유로 쉽게 설명하면
어노테이션은 역할을 알려주는 표지판이고,
스프링은 그 표지판을 읽고 알아서 일해주는 관리자예요.

예를 들어,

어노테이션	스프링의 반응
@Controller	"아, 이건 요청을 처리하는 클래스구나!"
@GetMapping("/hi")	"URL이 /hi면 이 함수 실행해줘야지!"
@Autowired	"이 객체는 내가 대신 만들어서 넣어줄게!"

✅ 요약
어노테이션 자체는 **그냥 표시(명찰)**일 뿐이에요.
실제로 그걸 보고 기능을 실행하는 건 스프링 프레임워크예요.
이건 리플렉션이라는 자바 기능 덕분에 가능하고,
개발자는 복잡한 코드를 직접 짜지 않아도 어노테이션만 붙이면 자동으로 처리되도록 도와주는 거예요.
